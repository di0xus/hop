## What Was Done

### Session 1: Initial Analysis & P0/P1 Performance Patches (Apr 30)

Committed to `v1.3.0-dev` (`bd0a086`):

| # | Change | File | Impact |
|---|--------|------|--------|
| P0-1 | Replaced thread-per-row regex with direct synchronous `re.is_match()`. Removed `mpsc` + `Duration` imports. | `score.rs` | Eliminated O(n) thread spawns on regex/negation queries |
| P0-2 | Added `last_computed_query` cache to picker. `compute_items` returns `(Vec, String)` and skips DB/score work when query hasn't changed. | `picker.rs` | Avoids full history reload + re-score on every keystroke |
| P1-1 | Wrapped `batch_upsert_indexed_dirs` in `BEGIN IMMEDIATE`/`COMMIT` transaction. Extracted `indexed_dir_params()` helper. | `db.rs` | True batch inserts for `hop reindex` |
| P1-3 | `prune_auto` now loads only `rowid+path` via 2 targeted SQL queries, then batch-deletes by rowid in chunks of 100. | `db.rs` | O(n) memory → O(1); N deletes → 1 batch per 100 |
| P1-4 | Removed dead `find_map`+`unwrap_or_else` pattern (threshold 20,000 unreachable). | `score.rs` | Cleaner code, no double iteration |
| P1-5 | Removed dead `IndexStats.batch` field and `flush_batch` method. | `index.rs` | Dead code gone |
